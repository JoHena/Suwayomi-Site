# ローカルソース

ローカルマンガを作成するには、以下の手順に従ってください。

1. [Suwayomi のデータディレクトリ](/ja/docs/data-directory)内にある `local` ディレクトリに、正しい構造でマンガを配置します。
2. **Local source** ソースでマンガにアクセスできるようになります。

チャプターを追加した場合は、手動でチャプターリストを更新する必要があります。

対応しているチャプター形式は、画像が入ったフォルダ(`.jpg`、`.png` など)、`ZIP`/`CBZ`、`RAR`/`CBR`、`EPUB` です。ただし、ディレクトリと `ZIP`/`CBZ` の方がパフォーマンスは良好です。

**注:** Suwayomi は **RAR** や **CBR** で圧縮されたチャプターにも対応していますが、**RAR5** 形式の **RAR**/**CBR** ファイルにはまだ対応していません。

**注:** **CBR** や **RAR** ファイルが動作しない場合は、対応形式に解凍・再圧縮する必要があるかもしれません。

## フォルダ構成

Suwayomi がローカルマンガを正しく処理するには、特定のフォルダ構成が必要です。ローカルマンガは `local` フォルダから読み込まれます。各マンガには `Manga` フォルダと `Chapter` フォルダが必要です。画像はチャプターフォルダに入れます。アーカイブファイルについては後述します。以下は例です。

```
local/
├── Manga title/
│   ├── Chapter 01 - Prologue/
│   │   ├── page01.png
│   │   ├── page02.png
│   │   ├── page03.png
│   │   └── ...
│   ├── Chapter 02 - A hero's journey/
│   │   ├── 01.jpg
│   │   ├── 02.jpg
│   │   ├── 03.jpg
│   │   └── ...
│   └── cover.jpg
└── Other Manga title/
    ├── cover.jpg
    ├── ch001/
    │   ├── 01 - cover.jpg
    │   ├── 02 - first page.jpeg
    │   ├── 03 - second.jpg
    │   └── ...
    └── ch002/
        ├── 001.jpg
        ├── 002.jpg
        ├── 003.jpg
        └── ...
```

**注:** チャプター名やページ名はアルファベット順・数字順に並ぶようにしてください。ゼロ埋めした番号を接頭辞にするのが通常もっとも確実です。

画像が入ったフォルダへのパスには、上記のようにマンガタイトルとチャプター名の両方を含める必要があります。

## アーカイブファイル

`ZIP`/`CBZ` などのアーカイブファイルには対応していますが、内部のフォルダ構成は無視されます。アーカイブ内のフォルダはすべて無視されます。アーカイブは `Manga` フォルダ内に配置する必要があり、そのファイル名が `Chapter` のタイトルになります。フォルダ構成にかかわらず、アーカイブ内のすべての画像がそのチャプターのページになります。

```
local/
└── Manga title/
    ├── ch1.zip
    ├── ch2.zip
    └── cover.jpg
```

## 応用

### ローカルマンガの詳細情報を編集する

ローカルマンガに詳細情報を追加することができます。他のカタログのマンガと同様に、作者、作画、あらすじ、ジャンルタグなどの情報を追加できます。

ローカルマンガと一緒に詳細情報を読み込ませるには、json ファイルを作成する必要があります。ファイル名は自由ですが、**Manga** フォルダ内に配置する必要があります。標準的なファイル名は `details.json` です。このファイルには `JSON` 形式でマンガの詳細情報を記述します。ファイルの作成方法は以下の例を参考にしてください。ファイルを配置すると、マンガを最初に開いたときにアプリがデータを読み込みます。あるいは、下にプルして詳細情報を再読み込みすることもできます。

以下の例をコピーして、必要に応じて内容を編集してください。
```json
{
	"title": "Example Title",
	"author": "Example Author",
	"artist": "Example Artist",
	"description": "Example Description",
	"genre": ["genre 1", "genre 2", "etc"],
	"status": "0",
	"_status values": ["0 = Unknown", "1 = Ongoing", "2 = Completed", "3 = Licensed"]
}
```

### カスタムカバー画像を使用する

各ローカルマンガにカスタム画像をカバーとして使用することもできます。

そのためには、`cover.jpg` という名前の画像ファイルをマンガフォルダの直下に配置するだけです。アプリはその画像をローカルソース一覧のカバーとして使用します。
